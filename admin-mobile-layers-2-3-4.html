Admin Mobile Direction

Layers 2, 3, and 4 should feel like one guided workspace.

The mobile admin flow should move from browse to focus to act without changing visual language every step. Instead of separate islands, each layer below uses the same structure: context header, sticky control stack, compact list rows, and one dominant action path.

What Changes
Panel 2 becomes a coherent browsing workspace: institute summary, mode switch, search, program filter, and class rows live inside one control rhythm.
Panel 3 becomes a focused bridge layer: one selected class or teacher, compact action chips, and a clean list for the next drilldown instead of oversized detached blocks.
Panel 4 becomes a true mobile detail view: sticky date filters, visible context, KPI strip, and a beautiful entry timeline with topics, slots, duration, and notes.
Delete/export actions stop shouting from every card. They move into calmer action chips or menus, so the information hierarchy becomes much easier to scan on a phone.
